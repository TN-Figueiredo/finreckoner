## What worked

- **Fresh subagent per task** — Claude orchestrator dispatched focused prompts with explicit scope, file boundaries, and self-review checklists. 40+ implementer/reviewer subagent invocations completed cleanly.
- **Wave-based parallelism** — after dependencies cleared, 3-4 agents in parallel on disjoint files (e.g., M0.4 Tasks 29+30+31 simultaneously).
- **Plan rigor pre-execution** — iterative audits pushed plan to 98/100 before code started; most subagent tasks were verbatim-copy from plan with zero ambiguity.
- **TDD discipline** — every unit task wrote failing test → implementation → pass → commit. Property-based tests (fast-check) caught several bracket-boundary edge cases pre-publish.
- **Small fixes applied inline** vs dispatching subagent — saved ~30-40min total on trivial 3-line fixes (Task 1 quality, ICU regex, sideEffects flag).

## What didn't

- **Publish workflow required 3 fixes** before succeeding (job-level env, .npmrc authToken line, fresh secret set). Plan's publish.yml was incomplete.
- **Bundle size budgets wrong by 3-4x** — plan estimated 3-10 KB per subpath; reality is 11-14 KB because tsup `splitting: false` + `external: decimal.js` duplicates primitives across subpaths.
- **Plan had a broken MoneyBrand design** — `declare const` type-only symbol used in runtime object literal causes ReferenceError. Implementer caught it; refactored to pure type-only brand post-audit.
- **Golden snapshots were self-referential** — the plan's YMYL note flagged this risk but the initial generation still ended up tautological. Added anchor verification + meta-test.
- **Wise cookie duration not validated in S0** (pre-flight user-manual step skipped). Slip to S1.
- **ICU version incompatibility** between Node 20 (CI) and Node 25 (dev) — one test regex needed relaxing.
